Phased Array Focal Law Calculator - User Manual

This document combines the technical instructions and the theoretical background for the calculator.

Part 1: Instructions

1. Overview

This tool calculates focal laws (delays) for ultrasonic phased array testing. It supports linear probes, wedges, and planar interfaces.

2. Python Usage

Launching the GUI:

python python/app.py

Features:

  • Probe/Wedge/Material: Configure your setup in the left panel.
  • Focus Modes:
    • Constant Depth: Focus at a fixed Z depth.
    • Vertical Line: Focus at a fixed X distance (useful for welds).
    • Constant Sound Path: Focus at a fixed radius from the entry point.
  • Wave Type: Select between Longitudinal and Shear wave velocities.
  • Visualization: The "Ray Tracing" tab provides a live view of the beams, wedge, and component.

3. MATLAB Usage

  1. Open MATLAB.
  2. Run PhasedArrayGUI in the command window (from the matlab/ folder).
  3. The interface mirrors the Python functionality.

Part 2: Theoretical Background

1. Fundamental Principle: Phasing

Phased Array Ultrasonic Testing (PAUT) relies on constructive interference. By firing individual elements (\(e_1, e_2, ..., e_n\)) at slightly different times, we can ensure that the wavefronts from all elements arrive at a specific point in space (the Focal Point) at the exact same moment.

The required Delay (\(\Delta t_i\)) for an element \(i\) is calculated as:

\[ \Delta t_i = T_{max} - T_{flight}^{(i)} \]

Where:

  • \(T_{flight}^{(i)}\) is the time of flight for sound to travel from Element \(i\) to the Focal Point.
  • \(T_{max}\) is the maximum time of flight across all elements in the array.

2. Geometry & Coordinate System

The calculator uses a global 2D Cartesian coordinate system \((X, Z)\):

  • Interface (Surface): Defined as the plane \(Z = 0\).
  • Component: Located at \(Z \ge 0\).
  • Wedge / Probe: Located at \(Z < 0\).

3. Ray Tracing: The Refraction Problem

The core challenge is calculating the **Time of Flight** for a ray traveling from an element in the wedge (Velocity \(v_1\)) to a target point in the component (Velocity \(v_2\)). Because the speed of sound changes at the interface, the ray bends according to Snell's Law:

\[ \frac{\sin(\alpha)}{v_1} = \frac{\sin(\beta)}{v_2} \]

However, finding the interface crossing point \(P(x, 0)\) for a fixed source and target is non-trivial. This calculator uses an Exact Analytical Solution based on Fermat's Principle (Minimum Time).

Fermat's Principle Solver

The total time function is:

\[ T(x) = \frac{\sqrt{(x - x_1)^2 + z_1^2}}{v_1} + \frac{\sqrt{(x_2 - x)^2 + z_2^2}}{v_2} \]

Minimizing this function leads to a 4th-degree (Quartic) Polynomial:

\[ ax^4 + bx^3 + cx^2 + dx + e = 0 \]

The calculator solves this polynomial to find the exact entry point.

4. Beam Steering Logic

To steer a beam to angle \(\beta\) in the material:

  1. Calculate the required incident angle \(\alpha\) using Snell's Law in reverse: \[ \alpha = \arcsin\left( \frac{v_1}{v_2} \sin(\beta) \right) \]
  2. Trace a ray from the **center** of the probe with angle \(\alpha\) to find the central entry point.
  3. Define the exact Focal Point \((x_f, z_f)\) along this ray path based on the chosen Focus Mode (Depth, Vertical, or Sound Path).
